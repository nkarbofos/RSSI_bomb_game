## Сборка проекта

Из корня репозитория:

```bash
cmake -S . -B build
cmake --build build -j 4
```

В результате будут собраны, в том числе:

- исполняемый файл сервера: `build/server/rssi_bomb_server_app`
- демонстрационный консольный клиент: `build/client_demo`

---

## Запуск сервера

```bash
./build/server/rssi_bomb_server_app            # порт по умолчанию 3000
# или
./build/server/rssi_bomb_server_app 3000       # явное указание порта
```

При успешном запуске в консоль выводится:

```text
rssi_bomb_server listening on port 3000
```

Сервер не завершается сам по себе: он ждёт входящих TCP‑подключений и обрабатывает команды от клиентов асинхронно.

Остановить сервер можно обычным `Ctrl+C` или сигналом `kill` по PID.

## Быстрый пример с двумя `nc`

Откройте три терминала:

1. В первом — сервер:

```bash
./build/server/rssi_bomb_server_app 3000
```

2. Во втором — клиент A (seeker, ищущий):

```bash
nc 127.0.0.1 3000
```

3. В третьем — клиент B (hider, прячущийся):

```bash
nc 127.0.0.1 3000
```

### Шаг 1. Создание лобби (seeker)

В терминале seeker:

```text
CREATE_LOBBY
```

Ответ сервера:

```text
INVITE ABC123
```

Сохраните `ABC123` — это код приглашения для второго игрока.

### Шаг 2. Подключение hider по коду

В терминале hider:

```text
JOIN_LOBBY ABC123
```

Оба клиента (seeker и hider) получат сообщение:

```text
GAME_START 10 10 30
```

Здесь:

- `10 10` — размер поля (ширина × высота)
- `30` — время фазы расстановки в секундах

### Шаг 3. Расстановка передатчиков (hider)

В терминале hider:

```text
PLACE_TRANSMITTERS 1 1 2 2 3 3
```

Оба клиента получат:

```text
PLACEMENT_DONE txReal=<1,1;2,2;3,3>
```

Если hider не успеет отправить команду за 30 секунд, сервер расставит передатчики случайно и также отправит `PLACEMENT_DONE` с фактическими координатами.

### Шаг 4. Ходы seeker’а

После завершения фазы расстановки начинается фаза поиска. Seeker может каждый ход выполнять одну команду:

- `GRADIENT_STEP`
- `MOVE_RECEIVER <idx> <x> <y>`
- `ADD_RECEIVER <x> <y>`
- `REMOVE_RECEIVER <idx>`
- `DONE`

После каждой корректной команды (кроме `DONE`) сервер возвращает состояние:

```text
STATE turn=1/20 rxpos=<5,5;8,3;4,1> txest=<1,2;2,3;7,7> rssi=<-45.20,-42.76,-39.33|-47.36,-45.82,-44.15|-39.83,-37.37,-37.37>
```

Где:

- `turn=1/20` — номер хода seeker’а и максимум ходов (20)
- `rxpos=<...>` — координаты всех приёмников
- `txest=<...>` — текущие оценки координат передатчиков (результат градиентного спуска)
- `rssi=<...>` — матрица RSSI:
  - строки разделены `|` и соответствуют приёмникам в порядке `rxpos`
  - в строке 3 значения — RSSI для 3 передатчиков в порядке `tx0..tx2`

#### Примеры команд seeker’а

- Одна итерация градиентного спуска:

  ```text
  GRADIENT_STEP
  ```

- Переместить приёмник `0` в клетку `(4,7)`:

  ```text
  MOVE_RECEIVER 0 4 7
  ```

- Добавить новый приёмник в клетку `(2,3)`:

  ```text
  ADD_RECEIVER 2 3
  ```

- Удалить приёмник с индексом `1`:

  ```text
  REMOVE_RECEIVER 1
  ```

Если действие недопустимо (например, выход за границы поля или попытка поставить приёмник на занятую клетку), сервер вернёт:

```text
ERROR <описание_ошибки>
```

### Шаг 5. Завершение игры

Игра завершается:

- по команде seeker’а:

  ```text
  DONE
  ```

- либо автоматически, после 20 ходов

В обоих случаях сервер отправляет итоговое сообщение:

```text
GAME_FINISHED txReal=<1,1;2,2;3,3> txEst=<1,2;2,3;7,7> mean_error_cells=1.234
```

Где:

- `txReal` — реальные координаты передатчиков
- `txEst` — финальные оценки координат
- `mean_error_cells` — средняя ошибка локализации в клетках

## Сводка команд протокола

### Для seeker (ищущий)

- `CREATE_LOBBY`
- `MOVE_RECEIVER <idx> <x> <y>`
- `ADD_RECEIVER <x> <y>`
- `REMOVE_RECEIVER <idx>`
- `GRADIENT_STEP`
- `DONE`

### Для hider (прячущийся)

- `JOIN_LOBBY <invite_code>`
- `PLACE_TRANSMITTERS <x1> <y1> <x2> <y2> <x3> <y3>`

### Ответы сервера

- `INVITE <invite_code>`
- `GAME_START <gridW> <gridH> <placementSeconds>`
- `PLACEMENT_DONE txReal=<x1,y1;...;x3,y3>`
- `STATE turn=<t>/<20> rxpos=<x,y;...> txest=<x,y;...> rssi=<row0>|<row1>|...`
- `GAME_FINISHED txReal=<...> txEst=<...> mean_error_cells=<value>`
- `ERROR <message>`
